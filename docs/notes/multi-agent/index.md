# 多 Agent 任务分发与协作原理与实践

深入讲解多 Agent 协作架构的两种实现方案：进程内协作与分布式任务分发。

## 目录

- [进程内协作方案](/notes/multi-agent/proceed) — 基于 Spring AI Alibaba Agent Framework，适合单体应用快速上线
- [分布式任务分发方案](/notes/multi-agent/distributed) — 基于 RocketMQ 异步解耦，适合微服务高并发场景

## 方案对比

| 维度 | 进程内（SubAgentInterceptor） | 分布式（+ 消息队列） |
|------|-------------------------------|---------------------|
| **适用规模** | 单体应用，QPS < 100 | 微服务，QPS > 1000 |
| **延迟** | 毫秒级 | 秒级（网络 + MQ） |
| **故障隔离** | 弱（共享 JVM） | 强（进程隔离） |
| **独立扩缩容** | 不支持 | 支持（按 Agent 独立扩容） |
| **长耗时任务** | 受 HTTP 超时限制 | 无限制（异步解耦） |

**推荐策略：** 初期用进程内方案快速上线 → 混合模式（短任务进程内，长任务走 MQ） → 全面分布式方案。
