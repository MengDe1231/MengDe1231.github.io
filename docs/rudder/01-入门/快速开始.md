# 快速开始

## 安装

### 全局安装（推荐）

```bash
npm install -g @mengde1231/rudder@latest
```

安装完成后验证：

```bash
rudder --version
```

### 环境要求

- **Node.js** >= 18
- **Python** >= 3.9

## 初始化项目

在已有项目中运行：

```bash
# 初始化所有平台（14 个平台全部配置）
rudder init -u your-name

# 或者只初始化你实际使用的平台（推荐，干净）
rudder init --claude --cursor --codex -u your-name
```

`-u your-name` 会设置开发者身份，后续所有 AI session 的工作日志都归集在这个名字下。

### 初始化后目录结构

```
your-project/
├── .rudder/                    # Rudder 工作流核心目录
│   ├── config.yaml             # 团队共享配置（git 跟踪）
│   ├── config_local.yml        # 个人本地覆盖（git 忽略）
│   ├── workflow.md             # 工作流定义
│   ├── .gitignore
│   ├── scripts/                # Python/Bash 脚本
│   ├── spec/                   # 编码规范模板
│   ├── tasks/                  # 任务目录
│   └── workspace/              # 开发者工作区
├── .claude/                    # Claude Code 配置（如已配置）
├── .cursor/                    # Cursor 配置（如已配置）
├── .codex/                     # Codex 配置（如已配置）
└── ...其他平台目录
```

## 更新配置

当 Rudder CLI 升级后，在项目根目录运行：

```bash
rudder update
```

这会检查 CLI 版本与项目记录版本的差异，并自动更新模板文件。

## 第一次使用

初始化完成后，打开你使用的 AI 编程平台（如 Claude Code），开始新对话即可。Rudder 会自动注入：

1. **Spec 规范** — `.rudder/spec/` 里的编码指南
2. **工具路径** — `.rudder/config_local.yml` 里配置的工具路径
3. **工作流状态** — 当前处于哪个阶段（Plan / Implement / Verify / Finish）

## 版本说明

当前公开版本从 `0.5.30` 开始。Rudder 诞生于 5 月，之前本地迭代了很多版本，因此主版本定为 0.5。
