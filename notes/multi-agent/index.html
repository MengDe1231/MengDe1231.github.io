多 Agent 任务分发与协作原理与实践 ​

深入讲解多 Agent 协作架构的两种实现方案：进程内协作与分布式任务分发。

目录 ​

  • 进程内协作方案 — 基于 Spring AI Alibaba Agent Framework，适合单体应用快速上线
  • 分布式任务分发方案 — 基于 RocketMQ 异步解耦，适合微服务高并发场景

方案对比 ​

维度进程内（SubAgentInterceptor）分布式（+ 消息队列）
适用规模单体应用，QPS < 100微服务，QPS > 1000
延迟毫秒级秒级（网络 + MQ）
故障隔离弱（共享 JVM）强（进程隔离）
独立扩缩容不支持支持（按 Agent 独立扩容）
长耗时任务受 HTTP 超时限制无限制（异步解耦）

推荐策略： 初期用进程内方案快速上线 → 混合模式（短任务进程内，长任务走 MQ） → 全面分布式方案。
